# How to add screenshots

Drop the PNGs here with **these exact names** (the README already references them):

| File | What to show |
|------|--------------|
| `dashboard.png` | The main dashboard with events/charts (DEMO data) |
| `nis2-report.png` | The reports / NIS2 compliance page |
| `alerts.png` | The alert management page |
| `mfa.png` | The Profile page with MFA / QR code |

## Capture safely

1. Use the dashboard with **demo or test data** — never real customer traffic.
2. Before the screenshot, make sure **none of these are visible**:
   - Real IPs (Expertmode, customers, or real attackers)
   - Credentials, tokens, secrets
   - Real customer or organisation names
3. If needed, blur/redact sensitive areas in an image editor.

## Capture on Windows

- **Snipping Tool** (Win + Shift + S) → select the area → paste into an editor
  → save as PNG with the correct name.

## Upload via the GitHub site

1. In the repo, browse to the `docs/img/` folder
2. **"Add file"** → **"Upload files"**
3. Drag the PNGs → **"Commit changes"**

The images then appear automatically in the README.

> Until you upload them, GitHub shows a broken-image icon — that's normal.
> You can delete this file afterwards.
