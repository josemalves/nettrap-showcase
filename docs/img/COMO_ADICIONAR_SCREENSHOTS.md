# Como adicionar screenshots

Coloca aqui os PNG com **estes nomes exatos** (o README já os referencia):

| Ficheiro | O que mostrar |
|----------|---------------|
| `dashboard.png` | O dashboard principal com eventos/gráficos (dados de DEMO) |
| `nis2-report.png` | A página de relatórios / conformidade NIS2 |
| `alerts.png` | A página de gestão de alertas |
| `mfa.png` | O ecrã de Perfil com o MFA / QR code |

## Como capturar em segurança

1. Usa o dashboard com **dados de demonstração ou de teste** — nunca com
   tráfego real de um cliente.
2. Antes do print, confirma que **não aparecem**:
   - IPs reais (da Expertmode, de clientes, ou de atacantes reais)
   - Credenciais, tokens, segredos
   - Nomes de clientes ou organizações reais
3. Se necessário, desfoca/tapa zonas sensíveis num editor de imagem.

## Como capturar no Windows

- **Ferramenta de Recorte** (Win + Shift + S) → seleciona a área → cola
  num editor → guarda como PNG com o nome certo.

## Como enviar para o GitHub (pelo site)

1. No repo da vitrine, navega até à pasta `docs/img/`
2. Botão **"Add file"** → **"Upload files"**
3. Arrasta os PNG → **"Commit changes"**

As imagens aparecem automaticamente no README.

> Quando não há imagem, o GitHub mostra um ícone de imagem partida — é
> normal até carregares os ficheiros. Podes apagar este `.md` depois.
